### 2. ✅ Flask Backend (app.py) - Фильтры и API

#### Улучшенный роут `/movie`
- Исключает 18+ контент
- Фильтрует аниме
- Блокирует эротику и порнографию

```python
@app.route('/movie')
def movie_list():
    # Фильтрует: 18+, аниме, взрослый контент
    banned_genres = ['аниме', 'anime', 'эротика', 'порно']
    has_banned_genres = any(b in genre_str for b in banned_genres)
```

#### Новые API endpoints

**`GET /api/recommendations?id=<movie_id>`**
- Возвращает 10 похожих фильмов по жанру
- Используется в watch.html для блока "Похожие фильмы"

**`GET /api/series/<series_id>`**
- Возвращает все эпизоды сериала поезд по сезонам
- Для интерфейса выбора сезонов/серий

**`POST /api/save-progress`**
- Сохраняет прогресс просмотра (0-99%)
- Для функции "Продолжить просмотр"

### 4. ✅ Live Search (live-search.js) - Умный поиск

#### Как работает:
1. Пользователь вводит более 2 символов в поиск
2. AJAX запросом ищет в БД (300мс debounce)
3. Показывает до 8 результатов с постерами

```javascript
// Автоматически интегрирован в base.html
<script src="{{ url_for('static', filename='js/live-search.js') }}"></script>
```

#### Результаты отображают:
- Постер фильма (40×60px)
- Название
- Год выпуска
- Рейтинг ⭐

### 6. ✅ Lazy Loading - Быстрая загрузка

#### Как настроить:
```html
<img src="" data-src="/static/posters/poster.jpg" loading="lazy">
```

**Что происходит:**
- IntersectionObserver следит за видимостью элементов
- При появлении в viewport подгружается изображение
- Экономит трафик и ускоряет первый load

---

### 7. ✅ Skeleton Loaders - До проекта загрузка

#### Использование в JavaScript:
```javascript
createSkeletons('rec-grid', 8);  // Создаст 8 skeleton карточек
```

**Результат:**
- Пока идет загрузка AJAX, показываются пульсирующие блоки
- Пользователь понимает, что контент загружается
- Исчезают когда приходят реальные данные

## 🎯 Ключевые особенности

### Архитектура
- ✅ **Не перепутано с base.html** - index.html чистый extend
- ✅ **Модульный код** - каждая функция отдельно

## 📝 Database структуры

### Для "Продолжить просмотр"
Требуется таблица (добавить при инициализации БД):
```sql
CREATE TABLE viewing_history (
    id INTEGER PRIMARY KEY,
    user_id INTEGER NOT NULL,
    movie_id INTEGER NOT NULL,
    progress REAL DEFAULT 0,  -- 0-99%
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, movie_id),
    FOREIGN KEY(user_id) REFERENCES users(id),
    FOREIGN KEY(movie_id) REFERENCES movies(id)
);
```

### Существующие таблицы задействованы:
- `movies` - основной контент (с фильтрацией)
- `series` + `episodes` - сериалы
- `favorites` - "Избранное" ❤️
- `tv_channels` - ТВ-каналы
